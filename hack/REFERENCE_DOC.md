# Flim.ai — API Reference

> **Base URL:** `http://localhost:3000`  
> **Protocol:** HTTP/1.1  
> **Body format:** JSON (unless stated otherwise)  
> **Auth mechanism:** `httpOnly` cookie `authentication-token` (JWT, 7-day expiry)

---

## Authentication

### `POST /auth/cookie` — Login

Issues a signed JWT as an `httpOnly` cookie on success.

**Request body**
```json
{ "name": "alice", "password": "hunter2" }
```

**Responses**

| Status | Body | Meaning |
|--------|------|---------|
| `200 OK` | `{ "message": "Logged in successfully" }` | Cookie set, ready to use |
| `400 Bad Request` | `{ "error": "name and password are required" }` | Missing fields |
| `401 Unauthorized` | `{ "error": "Invalid credentials" }` | Wrong name or password |

**Example (curl)**
```bash
curl -c cookies.txt -X POST http://localhost:3000/auth/cookie \
  -H "Content-Type: application/json" \
  -d '{"name":"alice","password":"hunter2"}'
```

---

### `POST /auth/register` — Register & Auto-login

Creates a new user record, hashes the password via Argon2, and issues a signed JWT cookie.

**Request body**
```json
{
  "name": "bob",
  "password": "hunter_bob",
  "email": "bob@flim.ai"
}
```

**Responses**

| Status | Body | Meaning |
|--------|------|---------|
| `201 Created` | `{ "message": "Registered and logged in successfully", "name": "bob" }` | User registered, cookie set |
| `400 Bad Request` | `{ "error": "name and password are required" }` | Missing fields |
| `400 Bad Request` | `{ "error": "Username already exists" }` | Username taken |
| `500 Internal Error` | `{ "error": "Failed to register user" }` | DB or server error |

**Example (curl)**
```bash
curl -c cookies.txt -X POST http://localhost:3000/auth/register \
  -H "Content-Type: application/json" \
  -d '{"name":"bob","password":"hunter_bob","email":"bob@flim.ai"}'
```

---

### `DELETE /auth/logout` — Logout

Clears the `authentication-token` cookie from the client. No auth required.

**Responses**

| Status | Body |
|--------|------|
| `200 OK` | `{ "message": "Logged out successfully" }` |

**Example (curl)**
```bash
curl -b cookies.txt -c cookies.txt -X DELETE http://localhost:3000/auth/logout
```

---

### `GET /valid` — Token check  *(auth required)*

Lightweight endpoint to verify the cookie is still valid.

**Responses**

| Status | Body |
|--------|------|
| `200 OK` | `{ "message": "Authorized", "user": "alice" }` |
| `403 Forbidden` | `{ "error": "No token provided" }` |
| `401 Unauthorized` | `{ "error": "Invalid or expired token" }` |

**Example (curl)**
```bash
curl -b cookies.txt http://localhost:3000/valid
```

## Food Management

All food uploads/snapping requests require a valid auth cookie.

---

### `POST /food/snap` and `POST /food/upload` — Upload & classify food  *(auth required)*

Accepts a `multipart/form-data` request with the image in the `food` or `image` field.  
The server saves the file to `./uploads/`, sends it to Gemini Vision for classification, captures location details, and persists the result to the database. Non-food items (e.g. chairs, toys) or spoiled, fungi-ish food are automatically rejected.

**Request fields (multipart/form-data)**

| Field | Type | Description |
|-------|------|-------------|
| `food` or `image` | File | The food image (JPG, PNG, WEBP, GIF) |
| `location` | string | Optional. JSON string representing coordinates, e.g. `'{"latitude": 12.9, "longitude": 77.5}'` |
| `from_location` | string | Optional. Alternative field for location. |
| `latitude` | string | Optional. Latitude coordinate field. |
| `longitude` | string | Optional. Longitude coordinate field. |

**Responses**

| Status | Body | Meaning |
|--------|------|---------|
| `200 OK` | `{ "id": 7, "name": "Biryani" }` | Classified and saved |
| `400 Bad Request` | `{ "error": "No file uploaded. Use field name 'food' or 'image'." }` | Missing file field |
| `400 Bad Request` | `{ "error": "Food upload rejected: Non-food or spoiled item" }` | Image classified as non-food, bad, or moldy |
| `401/403` | auth error | Cookie missing or invalid |
| `500` | `{ "error": "Failed to process food image" }` | Gemini or DB error |

**Example (curl)**
```bash
curl -b cookies.txt -X POST http://localhost:3000/food/snap \
  -F "food=@/path/to/biryani.jpg" \
  -F "location={\"latitude\":12.9716,\"longitude\":77.5946}"
```

> **Note:** The `name` in the response is the label generated by Gemini Vision.

---

### `GET /food/get?id=<integer>` — Fetch one food item

Returns the full database record for the given `id`.

**Query params**

| Param | Type | Required |
|-------|------|----------|
| `id` | integer | Yes |

**Response body (200 OK)**
```json
{
  "id": 7,
  "name": "Biryani",
  "time_posted": 1718437200,
  "publisher": "alice",
  "image": "food-1718437200000-123456789.jpg",
  "shelf_life": 86400,
  "from_location": "{\"latitude\":12.9716,\"longitude\":77.5946}",
  "buyer": "none",
  "buyer_location": "none"
}
```

| Status | Meaning |
|--------|---------|
| `200 OK` | Item found |
| `400 Bad Request` | `id` is not a valid integer |
| `404 Not Found` | No item with that id |

**Example (curl)**
```bash
curl http://localhost:3000/food/get?id=7
```

---

### `GET /food/available` — List available food

Returns every food item whose shelf life has **not yet expired**.

> Filter applied: `time_posted + shelf_life > now()` (all values in Unix seconds)

**Response body (200 OK)**
```json
[
  {
    "id": 7,
    "name": "Biryani",
    "time_posted": 1718437200,
    "publisher": "alice",
    "image": "food-1718437200000-123456789.jpg",
    "shelf_life": 86400,
    "from_location": "{\"latitude\":12.9716,\"longitude\":77.5946}",
    "buyer": "none",
    "buyer_location": "none"
  }
]
```

Returns an empty array `[]` if nothing is available.

**Example (curl)**
```bash
curl http://localhost:3000/food/available
```

---

### `GET /food/asset/:url` and `/uploads/:filename` — Static asset serving

Serves uploaded images statically. No authentication required.

**Example (curl)**
```bash
curl http://localhost:3000/food/asset/food-1718437200000-123456789.jpg
# Or directly from the uploads static route:
curl http://localhost:3000/uploads/food-1718437200000-123456789.jpg
```

## Order & Notification System

All order endpoints require a valid auth cookie.

---

### `POST /order/create` — Place a food order *(auth required)*

Places an order for a food item. Updates the buyer and buyer_location fields on the food item record, inserts a record in the orders table, and generates a notification message for the publisher. Order requests on expired food items are rejected.

**Request body**
```json
{
  "food_id": 7,
  "location": { "latitude": 12.9, "longitude": 77.6 }
}
```
*Note: The `location` field can also be a flat string, or separate `latitude` and `longitude` parameters.*

**Responses**

| Status | Body | Meaning |
|--------|------|---------|
| `201 Created` | `{ "message": "Order created successfully", "order_id": 1, "status": "pending" }` | Order placed |
| `400 Bad Request` | `{ "error": "food_id is required" }` | Missing food ID |
| `400 Bad Request` | `{ "error": "Cannot order: This food item has expired." }` | Cannot order expired food items |
| `404 Not Found` | `{ "error": "Food item not found" }` | Invalid food ID |
| `401/403` | auth error | Cookie missing or invalid |

**Example (curl)**
```bash
curl -b cookies.txt -X POST http://localhost:3000/order/create \
  -H "Content-Type: application/json" \
  -d '{"food_id": 7, "location": {"latitude": 12.9, "longitude": 77.6}}'
```

---

### `POST /order/respond` — Respond to order request *(auth required)*

Allows the publisher of the food item to accept or reject a pending order. Updates the order status and sends a response message back to the buyer.

**Request body**
```json
{
  "order_id": 1,
  "response": "yes"
}
```
*Note: Acceptable values for `response` are "yes", "accept", "no", "reject". Alternatively, you can pass `"status": "accepted"` or `"status": "rejected"`.*

**Responses**

| Status | Body | Meaning |
|--------|------|---------|
| `200 OK` | `{ "message": "Order status updated to accepted", "order_id": 1, "status": "accepted" }` | Status updated |
| `400 Bad Request` | `{ "error": "order_id is required" }` | Missing order ID |
| `400 Bad Request` | `{ "error": "Invalid response..." }` | Incorrect response value |
| `404 Not Found` | `{ "error": "Order not found" }` | Invalid order ID |
| `403 Forbidden` | `{ "error": "Unauthorized..." }` | Logged-in user is not the publisher of this item |

**Example (curl)**
```bash
curl -b cookies.txt -X POST http://localhost:3000/order/respond \
  -H "Content-Type: application/json" \
  -d '{"order_id": 1, "response": "yes"}'
```

---

### `GET /orders/my` — Fetch my orders *(auth required)*

Returns all orders where the current user is the buyer. Includes food item metadata (name, image, publisher).

**Query parameters**

| Param | Type | Required | Description |
|-------|------|----------|-------------|
| `format` | string | No | If set to `object`, returns `{ orders: [...], filled_stomachs_count: X }`. Default returns the array `[...]`. |

**Headers returned**
- `X-Filled-Stomachs-Count`: The system-wide count of accepted orders.

**Response body (default - Array)**
```json
[
  {
    "id": 1,
    "food_id": 7,
    "buyer": "bob",
    "buyer_location": "{\"latitude\":12.9,\"longitude\":77.6}",
    "status": "accepted",
    "time_ordered": 1718439900,
    "food_name": "Pizza",
    "food_image": "food-1718437200000-123456789.jpg",
    "publisher": "alice"
  }
]
```

**Example (curl)**
```bash
curl -b cookies.txt http://localhost:3000/orders/my
```

---

### `GET /orders/stats` — Fetch system order stats

Returns the overall statistics of successful orders.

**Response body (200 OK)**
```json
{
  "filled_stomachs_count": 5
}
```

---

### `GET /messages/my` — Inbox messages *(auth required)*

Returns messages/notifications sent to or received by the current user. Shows order requests ("Is <food> available?") and responses.

**Response body (200 OK)**
```json

  {
    "id": 2,
    "order_id": 1,
    "sender": "alice",

  },

    "id": 1,
    "order_id": 1,

    "recipient": "alice",
    "content": "Is Pizza available?",

    "type": "order_request",
    "time_sent": 1718439900,
    "food_name": "Pizza",
    "food_image": "food-1718437200000-123456789.jpg"
  }

```

**Example (curl)**
```bash
curl -b cookies.txt http://localhost:3000/messages/my
```

## Data Schema — `food_items`

| Column | Type | Description |
|--------|------|-------------|
| `id` | `INTEGER` | Primary key, auto-increment |
| `name` | `TEXT` | AI-classified food name |
| `time_posted` | `INTEGER` | Unix timestamp (seconds) of upload |
| `publisher` | `TEXT` | `name` of the authenticated user |
| `image` | `TEXT` | Filename inside `./uploads/` |
| `shelf_life` | `INTEGER` | Seconds until item expires (default: 86400) |
| `from_location` | `TEXT` | Location where food item is published (default: 'none') |
| `buyer` | `TEXT` | Username of the buyer who ordered the item (default: 'none') |
| `buyer_location` | `TEXT` | Location of the buyer who ordered the item (default: 'none') |

---

## Data Schema — `users`

| Column | Type | Description |
|--------|------|-------------|
| `id` | `INTEGER` | Primary key, auto-increment |
| `name` | `TEXT` | Unique username |
| `password` | `TEXT` | argon2id hash |
| `email` | `TEXT` | Optional email of the user |

---

## Data Schema — `orders`

| Column | Type | Description |
|--------|------|-------------|
| `id` | `INTEGER` | Primary key, auto-increment |
| `food_id` | `INTEGER` | ID of the ordered food item |
| `buyer` | `TEXT` | Username of the buyer |
| `buyer_location` | `TEXT` | Location of the buyer |
| `status` | `TEXT` | Status of order (`'pending'`, `'accepted'`, `'rejected'`) |
| `time_ordered` | `INTEGER` | Unix timestamp of order placement |

---

## Data Schema — `messages`

| Column | Type | Description |
|--------|------|-------------|
| `id` | `INTEGER` | Primary key, auto-increment |
| `order_id` | `INTEGER` | Associated order ID (optional) |
| `sender` | `TEXT` | Sender username |
| `recipient` | `TEXT` | Recipient username |
| `content` | `TEXT` | Message text content |
| `status` | `TEXT` | Read status (`'unread'`, `'read'`) |
| `type` | `TEXT` | Message type (`'text'`, `'order_request'`, `'order_response'`) |
| `time_sent` | `INTEGER` | Unix timestamp of transmission |

---

## Environment Variables

| Variable | Purpose |
|----------|---------|
| `JWT_SECRET` | Secret key for signing / verifying JWTs |
| `AI_API` | Google Gemini API key |

---

## Error Shape

All error responses follow the same JSON structure:

```json
{ "error": "Human-readable message" }
```

---

## Quick Start

```bash
# 1. Install dependencies
pnpm install

# 2. Ensure .env has JWT_SECRET and AI_API set

# 3. Seed a user (run once)
node seed.js

# 4. Start the dev server
pnpm dev
```
